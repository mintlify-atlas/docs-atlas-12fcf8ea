Get Vector installed and processing your first events in just a few minutes.

## Install Vector

<Tabs>
  <Tab title="Quick Install Script">
    The fastest way to get started:

    ```bash
    curl --proto '=https' --tlsv1.2 -sSfL https://sh.vector.dev | bash
    ```

    This script automatically detects your platform and installs the appropriate package.
  </Tab>

  <Tab title="Docker">
    Run Vector in a container:

    ```bash
    docker pull timberio/vector:latest-alpine
    docker run -v $(pwd)/vector.yaml:/etc/vector/vector.yaml:ro \
      timberio/vector:latest-alpine
    ```
  </Tab>

  <Tab title="Homebrew (macOS)">
    Install via Homebrew:

    ```bash
    brew tap vectordotdev/brew
    brew install vector
    ```
  </Tab>
</Tabs>

<Note>
  See the [Installation](/installation) guide for more installation methods including Kubernetes, APT, YUM, and systemd.
</Note>

## Create Your First Pipeline

Create a file named `vector.yaml` with this basic configuration:

```yaml vector.yaml
# Data source: Generate demo logs
sources:
  demo_logs:
    type: demo_logs
    format: syslog
    interval: 1

# Transform: Parse syslog format
transforms:
  parse_logs:
    type: remap
    inputs:
      - demo_logs
    source: |
      . = parse_syslog!(string!(.message))
      .timestamp = now()

# Data destination: Print to console
sinks:
  console:
    type: console
    inputs:
      - parse_logs
    encoding:
      codec: json
```

This configuration:
1. **Generates** demo syslog data every second
2. **Parses** the syslog format into structured JSON
3. **Outputs** the results to your console

## Run Vector

Start Vector with your configuration:

```bash
vector --config vector.yaml
```

You should see structured log events being printed to your console:

```json
{
  "appname": "benefritz",
  "facility": "local1",
  "hostname": "some.machine.com",
  "message": "Hello world",
  "msgid": "ID31",
  "severity": "notice",
  "timestamp": "2024-03-05T10:30:00.000Z"
}
```

<Tip>
  Press `Ctrl+C` to stop Vector.
</Tip>

## Common First Steps

<AccordionGroup>
  <Accordion title="Collect Real Logs">
    Replace the `demo_logs` source with a real source like `file`:

    ```yaml
    sources:
      app_logs:
        type: file
        include:
          - /var/log/app/*.log
    ```

    See [Sources](/reference/sources/overview) for all available sources.
  </Accordion>

  <Accordion title="Send to a Real Destination">
    Replace the `console` sink with a production destination:

    ```yaml
    sinks:
      elasticsearch:
        type: elasticsearch
        inputs:
          - parse_logs
        endpoint: "https://elasticsearch.example.com"
        index: "logs-%Y.%m.%d"
    ```

    See [Sinks](/reference/sinks/overview) for all available sinks.
  </Accordion>

  <Accordion title="Add Data Enrichment">
    Enhance your data with additional fields:

    ```yaml
    transforms:
      enrich:
        type: remap
        inputs:
          - parse_logs
        source: |
          .environment = "production"
          .datacenter = get_hostname()
          .app_version = "2.0.1"
    ```

    Learn more in the [VRL Guide](/guides/vrl-basics).
  </Accordion>

  <Accordion title="Validate Your Configuration">
    Check your configuration before deploying:

    ```bash
    vector validate vector.yaml
    ```

    See [Validation](/administration/validation) for more testing options.
  </Accordion>
</AccordionGroup>
