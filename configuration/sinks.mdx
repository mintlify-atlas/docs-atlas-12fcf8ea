Sinks are the output components in Vector. They send processed events to various destinations like databases, cloud services, files, and more.

## Sink Configuration

Sinks are configured in the `sinks` section:

```yaml
sinks:
  <sink_id>:
    type: <sink_type>
    inputs: [<source_or_transform_ids>]
    # Sink-specific options
```

### Common Sink Parameters

All sinks support these base configuration options:

<ParamField path="type" type="string" required>
  The type of the sink component.
</ParamField>

<ParamField path="inputs" type="array" required>
  Array of source or transform IDs to receive events from.
</ParamField>

<ParamField path="healthcheck" type="object">
  Healthcheck configuration for the sink.
  
  ```yaml
  healthcheck:
    enabled: true
    uri: "http://localhost:9200/_cluster/health"
  ```
</ParamField>

<ParamField path="buffer" type="object">
  Buffering configuration for the sink.
  
  ```yaml
  buffer:
    type: memory
    max_events: 500
    when_full: block
  ```
</ParamField>

<ParamField path="acknowledgements" type="boolean" default="false">
  Enable end-to-end acknowledgements for this sink.
</ParamField>

## Console Sink

Output events to stdout/stderr:

```yaml
sinks:
  console_output:
    type: console
    inputs: [parse_logs]
    encoding:
      codec: json
      json:
        pretty: true
```

<ParamField path="target" type="string" default="stdout">
  Output target: `stdout` or `stderr`.
</ParamField>

<ParamField path="encoding" type="object" required>
  How to encode events for output.
</ParamField>

<ParamField path="encoding.codec" type="string" required>
  Encoding format: `json`, `text`, `logfmt`, or `csv`.
</ParamField>

## File Sink

Write events to files:

```yaml
sinks:
  file_output:
    type: file
    inputs: [parse_logs]
    path: "/var/log/vector/output-%Y-%m-%d.log"
    encoding:
      codec: json
    compression: gzip
```

<ParamField path="path" type="string" required>
  File path template. Supports strftime specifiers for time-based rotation.
</ParamField>

<ParamField path="compression" type="string" default="none">
  Compression algorithm: `none`, `gzip`, or `zstd`.
</ParamField>

<ParamField path="idle_timeout_secs" type="integer" default="30">
  Close idle files after this many seconds.
</ParamField>

## Elasticsearch Sink

Send events to Elasticsearch:

```yaml
sinks:
  elasticsearch:
    type: elasticsearch
    inputs: [parse_logs]
    endpoint: "http://localhost:9200"
    bulk:
      index: "vector-%Y-%m-%d"
      action: index
    auth:
      strategy: basic
      user: "elastic"
      password: "${ELASTICSEARCH_PASSWORD}"
```

<ParamField path="endpoint" type="string" required>
  Elasticsearch endpoint URL.
</ParamField>

<ParamField path="bulk.index" type="string" required>
  Index name template. Supports strftime specifiers.
</ParamField>

<ParamField path="bulk.action" type="string" default="index">
  Bulk operation type: `index`, `create`, or `update`.
</ParamField>

<ParamField path="auth" type="object">
  Authentication configuration for Elasticsearch.
</ParamField>

## AWS S3 Sink

Upload events to Amazon S3:

```yaml
sinks:
  s3_archive:
    type: aws_s3
    inputs: [parse_logs]
    region: us-east-1
    bucket: log-archives
    key_prefix: "date=%Y-%m-%d/"
    compression: gzip
    encoding:
      codec: json
    batch:
      max_bytes: 10000000  # 10MB
      timeout_secs: 300
```

<ParamField path="region" type="string" required>
  AWS region where the bucket is located.
</ParamField>

<ParamField path="bucket" type="string" required>
  S3 bucket name.
</ParamField>

<ParamField path="key_prefix" type="string">
  Object key prefix. Supports strftime specifiers for partitioning.
</ParamField>

<ParamField path="compression" type="string" default="gzip">
  Compression algorithm: `none`, `gzip`, or `zstd`.
</ParamField>

<ParamField path="batch" type="object">
  Batching configuration for S3 uploads.
</ParamField>

## HTTP Sink

Send events to HTTP endpoints:

```yaml
sinks:
  http_endpoint:
    type: http
    inputs: [parse_logs]
    uri: "https://api.example.com/logs"
    method: post
    encoding:
      codec: json
    auth:
      strategy: bearer
      token: "${API_TOKEN}"
```

<ParamField path="uri" type="string" required>
  HTTP endpoint URL.
</ParamField>

<ParamField path="method" type="string" default="post">
  HTTP method: `post`, `put`, or `patch`.
</ParamField>

<ParamField path="headers" type="object">
  Custom HTTP headers to include.
  
  ```yaml
  headers:
    X-Custom-Header: "value"
    Content-Type: "application/json"
  ```
</ParamField>

## Kafka Sink

Produce events to Apache Kafka:

```yaml
sinks:
  kafka:
    type: kafka
    inputs: [parse_logs]
    bootstrap_servers: "localhost:9092"
    topic: logs
    key_field: host
    encoding:
      codec: json
```

<ParamField path="bootstrap_servers" type="string" required>
  Comma-separated list of Kafka bootstrap servers.
</ParamField>

<ParamField path="topic" type="string" required>
  Kafka topic to produce messages to.
</ParamField>

<ParamField path="key_field" type="string">
  Event field to use as the Kafka message key.
</ParamField>

## Prometheus Exporter Sink

Expose metrics for Prometheus scraping:

```yaml
sinks:
  prometheus_exporter:
    type: prometheus_exporter
    inputs: [app_metrics]
    address: "0.0.0.0:9598"
    default_namespace: vector
```

<ParamField path="address" type="string" required>
  Socket address to listen on for Prometheus scrapes.
</ParamField>

<ParamField path="default_namespace" type="string" default="vector">
  Default namespace for metrics without one.
</ParamField>

## Loki Sink

Send logs to Grafana Loki:

```yaml
sinks:
  loki:
    type: loki
    inputs: [parse_logs]
    endpoint: "http://localhost:3100"
    labels:
      environment: production
      host: "{{ host }}"
    encoding:
      codec: json
```

<ParamField path="endpoint" type="string" required>
  Loki endpoint URL.
</ParamField>

<ParamField path="labels" type="object" required>
  Label set to apply to all logs. Supports template syntax.
</ParamField>

## Datadog Logs Sink

Send logs to Datadog:

```yaml
sinks:
  datadog_logs:
    type: datadog_logs
    inputs: [parse_logs]
    default_api_key: "${DATADOG_API_KEY}"
    site: datadoghq.com
```

<ParamField path="default_api_key" type="string" required>
  Datadog API key for authentication.
</ParamField>

<ParamField path="site" type="string" default="datadoghq.com">
  Datadog site: `datadoghq.com`, `datadoghq.eu`, etc.
</ParamField>

## Multiple Sinks Example

Send data to multiple destinations:

```yaml
sources:
  app_logs:
    type: file
    include: ["/var/log/app/*.log"]

transforms:
  parse:
    type: remap
    inputs: [app_logs]
    source: |
      . = parse_json!(.message)
  
  route:
    type: route
    inputs: [parse]
    route:
      errors: '.level == "error"'
      metrics: '.type == "metric"'
      other: 'true'

sinks:
  # Error logs to Elasticsearch for analysis
  elasticsearch_errors:
    type: elasticsearch
    inputs: [route.errors]
    endpoint: "http://localhost:9200"
    bulk:
      index: "errors-%Y-%m-%d"
  
  # All logs to S3 for long-term storage
  s3_archive:
    type: aws_s3
    inputs: [parse]
    region: us-east-1
    bucket: log-archives
    key_prefix: "date=%Y-%m-%d/"
    compression: gzip
  
  # Metrics to Prometheus
  prometheus:
    type: prometheus_exporter
    inputs: [route.metrics]
    address: "0.0.0.0:9598"
  
  # Debug output to console
  console:
    type: console
    inputs: [route.other]
    encoding:
      codec: json
```

## Buffering

Configure how sinks buffer events:

```yaml
sinks:
  elasticsearch:
    type: elasticsearch
    inputs: [logs]
    endpoint: "http://localhost:9200"
    buffer:
      type: disk
      max_size: 268435488  # 256 MB
      when_full: block
```

<ParamField path="buffer.type" type="string" default="memory">
  Buffer type: `memory` or `disk`.
</ParamField>

<ParamField path="buffer.max_events" type="integer" default="500">
  Maximum number of events to buffer (memory buffer only).
</ParamField>

<ParamField path="buffer.max_size" type="integer">
  Maximum buffer size in bytes (disk buffer only).
</ParamField>

<ParamField path="buffer.when_full" type="string" default="block">
  Behavior when buffer is full: `block` or `drop_newest`.
</ParamField>

## Batching

Configure how sinks batch events:

```yaml
sinks:
  http_endpoint:
    type: http
    inputs: [logs]
    uri: "https://api.example.com/logs"
    batch:
      max_events: 100
      max_bytes: 1048576  # 1 MB
      timeout_secs: 10
```

<ParamField path="batch.max_events" type="integer">
  Maximum number of events per batch.
</ParamField>

<ParamField path="batch.max_bytes" type="integer">
  Maximum batch size in bytes.
</ParamField>

<ParamField path="batch.timeout_secs" type="integer">
  Maximum time to wait before sending a partial batch.
</ParamField>
